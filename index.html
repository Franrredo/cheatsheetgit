Cheatsheet GIT

Voy comenzar una pequeña guia recordatorio de Git para poder tenerla a mano algunos de estos comandos incluiran breves explicaciones.

Zonas de Git

  1. Directorio de trabajo(Actual)
  2. Área de preparación(addss)
  3. Directorio Git(Commit)

Configuración

  • git config --global user.name "nombre apellidos"
  • git config --global user.email "demo@demo.es"
  • git config --list muestra todas las configuraciones
  • git config --global core.editor eliges tu editor para el texto del cambio ejemplo Notepad(Windows)
  • git help "valor" muestra la ayuda respecto a un termino

Comandos Basicos

  • git init(inicia un repositorio)
  • git status(te dice el estado del repositorio si hay un archivo nuevo y no lo hemos agregado)
  • git add nombredearchivo(añade el archivo)
  • git add . (añade todos los archivos)
  • git commit -m "mensaje que queramos"(Así no nos aparece el editor de texto)
  • git commit -m "mensaje (en este caso pulsamos enter antes de cerrar las comillas y escribimos todo lo que deseemos para finalizar escribimos comillas)"
  • git commit -a -m(pasa a la zona de confimación todos los añadidos anteriormente y realiza el commit)
  • git commit (igual que el anterior pero abre el programa que elegimos en el core)
  • git log(Los commits que llevamos realizados)
  • git log --oneline(los commits pero en una linea)
  • git log --graph
  • git log --numerocomits (muestra la cantidad de últimos commits que quieres ver)
  • git diff(para ver la direncia con el commit anterior)
  • git diff --staged(la diferencia entre los archivos añadidos add y la modificación actual despues del add)
  • git reset HEAD nombre_de_archivo (saca el archivo de zona de preparación)
  • git add -A (agrega todos los que esten en areá de preparación)
  • git commit --amend (Editamos el mensaje del último commit)
  • git rm "nombre de archivo"(Elimina archivos add de nuestro directorio de trabajo, así no apareceran como no rastreados)
  • git checkout -- "archivo" (recupera el archivo borrado)
  • git mv "inicial" "final"(renombra el archivo)

.gitignore

Es un archivo .gitignore en la carpeta raíz , donde se ponen los archivos que no queremos seguir https://www.gitignore.io/ web donde te dan patrones según la programción que uses ejemplo Symfony
